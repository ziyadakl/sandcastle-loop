## False-negatives — slices that shipped CLEAN where a real regression was caught later

Two confirmed false-negatives, both on the same root cause:

1. **`d5c452caf fix(post-merge): apply simplify pass to Plaid backend across #308, #309 (per post-merge review)`** — `git log --format='%H%n%s%n%n%b' d5c452caf`. Post-merge reviewer flagged that both type:backend issues had been merged without the required `simplify` rubric pass. Commit applied two real `simplify` reductions:
   - `verify-webhook.ts`: ~30 lines of hand-rolled `p1363ToDer` machinery deleted; replaced with Node's built-in `crypto.verify({ dsaEncoding: "ieee-p1363" })` option. Equivalent security, less code.
   - `webhook-handler.ts`: body parsed twice (once for routing, once for audit-log); collapsed to a single parse.
     Critique verdict for both was `CRITIQUE_CLEAN`. Skill-discipline `missed: simplify` was the correct signal here; critique's silent abstention on `simplify` was a false-negative.

2. **`0ded14d47 fix(post-merge): record fixer skill-discipline pass across #303 (per post-merge review)`** — same shape, on #303. Reviewer explicitly noted "no functional defect" but still required the rubric pass. The fixer commit then ran a real `simplify` review on `get-rate.ts` (flagging two near-duplicate SELECT blocks). #303's critique was CLEAN. Same pattern: skill-discipline right, critique silently wrong.

No regressions found that trace back to #305, #307, #318, #319, #321 (#307 was caught at critique time, blocked merge, and was patched in `c669e6bc2 fix(plaid): drop premature linked-banks-menu copy in account picker`).

### Bottom-line

**Recommendation: mixed — adopt critique-as-gate into the master template, but DO NOT demote skill-discipline from a hard gate to telemetry. Run both.**

The evidence from the Plaid run splits cleanly. On UI-shaped issues (#307, #318, #319, #321) critique loaded substantive design rubrics (`impeccable`, `harden`, `adapt`, `polish`, `clarify`) and either passed them honestly or caught a real P1 (#307). On these, critique is the value add — it actually graded the diff against design vocabulary, where skill-discipline only counted tool calls. But on backend issues (#303, #308, #309) the project's `simplify` and `context7-docs` principles have no `SKILL.md` file, so critique silently abstained, returned CLEAN, and let two real `simplify` regressions ship — which post-merge reviewer then had to fix in `d5c452caf` and `0ded14d47`. Skill-discipline's WARN was the only signal that pointed at the right rubric on those slices. Adopt critique-as-gate for its design-rubric power, but keep skill-discipline blocking too (or write the missing `simplify`/`context7-docs` SKILL.md files so critique stops abstaining).
